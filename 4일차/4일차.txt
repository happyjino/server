역할과 기능의 차이
역할은 로그인 같은 인증이 필요한 서비스
기능은 단순하게 유틸리티들만! 인증이 필요없다!
역할은 기능과 동반설치해야되는 것이 대부분!!
역할은 인증하는 메커니즘!!
기능은 설치하면 끝~~~

이제 도메인 서버 만들자!
adds 호스트에다가!
도메인 하기 전에 active directory domain service 체크, dns 체크
필요한 기능 추가하겠습니까?
기능 - 원격 관리도구 체크 안되어있는게
위의 서버 역할을 추가 하면 자동으로 체크되어있다

윈도우 서버 2008까지는 도메인 서버를 설치시
dcpromo(domain control promotion)라는 명령어를 실행해서 도메인 설치마법사로 진행 했으나 2012버전부터 이 명령어는 사용 못하게 됨.

도메인 컨트롤러는 일종의 논리적인 울타리
윈도우에는 workgroup, domain이 존재
workgroup는 각자의 집, 개개인
domian은 회사, 부대, 학교 -> 중앙제어하는 사장, 부대장이 있다!
울타리 안에는 서비스하는 서버, 멤버의 컴퓨터들, 계정들이 들어가 있다.
이들이 모두 동기화 돼서 사용된다.
ex) 구글로 유튜브! 컴퓨터로 구글 로그인해서 유튜브! 우영우 영상본다
핸드폰으로 들어가면 우영우 영상이 쭉 나온다! 동기화 때문에~~

도메인 컨트롤러는 모든 도메인에 1개 이상!! 설치되어야 한다~
반드시! 도메인 서버에는 도메인 컨트롤러가 1개 이상 설치되어야 한다!
