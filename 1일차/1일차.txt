server란?
사용자 요청에 의한 서비스 제공
서버란 갖고 있는 자원을 서비스를 요청한 클라이언트에게 제공해주는 장치
받는건 딱하나? -> 요청!
갖고 있는 자원 => s/w, h/w

os(운영체제) : 사용자 h/w, s/w의 중간 매개체 역할
컴퓨터에게 명령을 내리면 그냥은 못 알아먹는다. -> 컴퓨터에게 맞게 번역할 터미널 필요 -> shell
shell에서 커널로 보낸다! -> 명령 수행
sw 구동 시키는 역할도 한다!

클라이언트 쪽에서 사용하는 운영체제만을 사용하다가
서버 쪽에서 사용하는 윈도우 서버, 리눅스를 배운다!!!

리눅스 만든 사람 : 리처드 스톨먼, 리누스 토발즈

윈도우의 논리적 2개의 그룹
workgroup - 동일한 네트워크에서의 그룹화, 파일을 공유하는 정도의 서비스만 가능(롤-내 챔프만)
domain - 광범위한 네트워크에서 하나로 그룹화 가능, 중앙 집중 제어가 가능(스타 - 여러 유닛을)

파일 시스템 : 저장 공간 제공, 데이터 전달

프린트 시스템 : 딱히 필요없음. 프린트기가 대체

터미널 : 목적이 있다!
